# ระบบสะสมแต้ม (Loyalty Points) — โจทย์ตั้งต้น

## ที่มา
พี่สั่ง (Inline mode): "ช่วยประเมินระบบสะสมแต้มให้หน่อย"

## Scope ที่ตกลงกับพี่ (จากคำถามเลือกได้ 2026-06-20)
- **ช่องทางลูกค้า**: หน้าร้าน/พนักงานกดให้เท่านั้น (ลูกค้าไม่มี self-service)
  → ต้องสร้าง Staff Terminal (web responsive) ให้พนักงานใช้หน้าร้าน
- **โครงสร้างธุรกิจ**: Multi-tenant SaaS — รองรับหลายแบรนด์บนระบบเดียว
- **กฎสะสม-แลกแต้ม**: ครบ 4 ส่วน
  - สะสม + แลกรางวัลพื้นฐาน (core)
  - ระดับสมาชิก (Tier)
  - โปรโมชัน/แคมเปญ (แต้มคูณ/โบนัส)
  - แนะนำเพื่อน (Referral)
- **Integration**: standalone — ยังไม่ต่อ POS/payment/แจ้งเตือนภายนอกในเฟสนี้

## สมมติฐานสำคัญ (ขุดต่อ)
- ลูกค้าไม่มีหน้าใช้เอง → ระบุตัวสมาชิกด้วยเบอร์โทรที่ staff terminal
- Referral: พนักงานกรอกรหัส/เบอร์ผู้แนะนำตอนสมัครสมาชิกใหม่ (เพราะ POS-only)
- บันทึกยอดซื้อด้วยการคีย์มือที่ terminal เพื่อคำนวณแต้ม (ไม่ต่อ POS เดิม)
- โมเดลเก็บค่าบริการร้าน (billing) ยังไม่อยู่ใน scope รอบนี้ — เสนอแยก
